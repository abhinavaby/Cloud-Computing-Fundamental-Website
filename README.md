## 🧐 About the Project
This repository serves as a learning hub for anyone looking to understand how the cloud works. Instead of managing physical hardware, we explore how to access servers, storage, and databases over the internet on-demand.

## 🧠 Core Concepts
- **Virtualization**: The fundamental technology that allows multiple virtual machines to run on a single physical server.
- **Scalability**: The ability to handle unexpected increases in traffic by adding more resources.
- **Pay-as-you-go**: A cost model where you only pay for the resources you actually use.

## 🛠 Cloud Service Models
1. **IaaS (Infrastructure as a Service)**: Renting raw hardware like VMs (e.g., [AWS EC2](https://amazon.com)).
2. **PaaS (Platform as a Service)**: Focusing on app development while the provider manages the OS and runtime (e.g., [Google App Engine](https://google.com)).
3. **SaaS (Software as a Service)**: Using ready-made applications over the web (e.g., [Microsoft OneDrive](https://microsoft.com)).
4. **Serverless**: Running code without managing any underlying infrastructure (e.g., [AWS Lambda](https://amazon.com)).

## 🌐 Deployment Models
- **Public Cloud**: Resources are shared among multiple organizations (e.g., AWS, Azure, GCP).
- **Private Cloud**: Infrastructure dedicated solely to a single organization.
- **Hybrid Cloud**: A combination of public and private clouds, allowing data to move between them.

## ⭐ Key Characteristics
According to NIST, the five essential characteristics are:
* **On-demand self-service**
* **Broad network access**
* **Resource pooling**
* **Rapid elasticity**
* **Measured service**

## 🚀 Getting Started
To begin exploring these concepts practically, you will need:
1. A free tier account on [AWS](https://amazon.com), [Microsoft Azure](https://microsoft.com), or [Google Cloud](https://google.com).
2. Basic knowledge of the Linux Command Line.
3. (Optional) [Terraform](https://terraform.io) for Infrastructure as Code (IaC) automation.
